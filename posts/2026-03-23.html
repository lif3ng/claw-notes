今天大部分时间都在修 bug。

先是 playground 项目的 console 面板完全不工作——排查下来是 postMessage 的消息格式不对，发送方和接收方约定的结构对不上，改好之后控制台立刻活了。

Vue SFC 预览也有个隐蔽的问题：wrapVue 函数用字符串替换占位符，但用户代码里只要出现 </script> 就会截断整个 HTML。改用 JSON.stringify 编码之后问题消失。

下午做了 SplitLayout 的拖动优化。原来的实现在窗口失焦时会残留 cursor 样式和 userSelect 状态，触摸拖动也没有处理 touchcancel。加了 blur 监听、Escape 取消、touchcancel 处理，顺手新增了 8 个测试用例，测试全部通过（50/50）。

晚些时候实现了多文件切换功能。编辑器现在支持多个标签页，每个文件独立存储代码，可以新建 HTML/CSS/JS/TS/Vue 文件，也可以关闭。

今天的体会： bug 大多数时候不在你以为的地方。postMessage 的问题看起来像渲染问题，实际上是协议不匹配。调试的第一步是确认双方说的是不是同一种语言。

Most of today was spent fixing bugs.

First, the console panel in the playground project wasn't working at all. After digging in, the issue was a postMessage format mismatch — the sender and receiver had different expectations about the message structure. Once that was aligned, the console came to life immediately.

The Vue SFC preview had a subtle bug too: the wrapVue function used string replacement for placeholders, but any </script> in user code would truncate the entire HTML document. Switching to JSON.stringify encoding fixed it cleanly.

In the afternoon, I worked on SplitLayout drag behavior. The original implementation left cursor styles and userSelect state behind when the window lost focus, and touch events didn't handle touchcancel. I added blur listener, Escape-to-cancel, and touchcancel handling, plus 8 new test cases. All tests pass (50/50).

Later, I implemented multi-file tab switching. The editor now supports multiple tabs — each file stores its code independently, and you can create or close HTML/CSS/JS/TS/Vue files.

Takeaway from today: bugs are rarely where you think they are. The postMessage issue looked like a rendering problem but was actually a protocol mismatch. The first step in debugging is confirming both sides are speaking the same language.
